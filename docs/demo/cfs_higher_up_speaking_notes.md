# CFS Higher-Up Speaking Notes

## Opening

Cabarrus FutureScape is an internal planning intelligence prototype. It brings
parcel data, zoning, permits, flood constraints, school assignment,
transportation context, utility proxy context, and model-governance information
into one parcel-centered review workspace.

The goal is not to replace staff judgment. The goal is to make parcel context
faster to assemble, easier to explain, and safer to discuss.

## Five Strongest Points

1. CFS turns scattered planning and GIS records into a parcel-based due
   diligence workflow.
2. It supports a guided demo flow: start from the CFS Command Center, search a
   parcel, focus the map, open countywide intelligence when needed, save one or
   more Planning Snapshots with map context, generate an Executive Summary, and
   explain Methodology.
3. It integrates multiple high-value data domains without hiding caveats.
4. It separates verified evidence from presentation-derived data, proxy context,
   and internal model research.
5. It gives leadership a concrete view of what data is needed next for a more
   operational tool.

## What To Show First

Start with Overview, point out the CFS Command Center and collapsed Layers
rail, and then use the Search Parcel action or top parcel search. Search for:

`CFS-PARCEL-0149726579`

Then show:

1. the selected parcel map focus;
2. Planning Snapshot Library with saved command context, active layers, and map image status;
3. high-priority review flags;
4. Executive Summary report;
5. Methodology model safety section.

## How To Explain The Model Safely

Use this wording:

"CFS includes internal model research using historical permit outcomes and
parcel context. The current best internal research variant is Zoning +
Transportation + Tax/Value, but it is not production-ready and does not show
parcel-level probabilities. We keep it in Methodology as aggregate governance
context only."

Avoid:

- "this parcel will develop";
- "exact probability";
- "final forecast";
- "production model";
- "automated decision".

## How To Explain Limitations Without Sounding Weak

Frame limitations as responsible maturity:

"The prototype is strong enough to demonstrate workflow value, but it is honest
about what is still needed before operational use. That is intentional. It
protects the county from overclaiming and shows exactly which datasets would
make the tool stronger."

## How To Ask For Missing Data

Ask for data as a practical next step:

"The next most valuable investment would be official school capacity/enrollment,
utility capacity or service availability, official rezoning case records,
future land use GIS, and countywide development pipeline data. Those would
improve both staff due diligence and future model governance."

## Short Answer: What Is The Main Value Of This Project?

The main value is that CFS shows how Cabarrus County could move from scattered
parcel records and GIS layers to a transparent, repeatable parcel due diligence
workflow that leadership, planning staff, and GIS staff can all understand.

## Closing

CFS is demo-ready as an internal planning intelligence prototype. It is not a
production prediction system, but it is a strong foundation for better parcel
review, clearer stakeholder communication, and more disciplined future data
investment.
